В конфигурации «Аванкор: Доверительное управление» в рамках единого плана счетов реализован механизм автоматического ведения 20-значных лицевых счетов по настройкам, задаваемым в регистре «Настройка аналитики лицевых счетов НФО». Настройки аналитики можно задавать в разрезе каждого портфеля. При изменении настройки аналитик для одного из парных счетов, для второго настройки точно так же меняются автоматически.

Справочник «Лицевые счета» доступен из меню: «Прочая информация» → «См. также» → «Лицевые счета ЕПС».

На форме списка лицевых счетов имеются кнопки:

§   Лицевые счета с ошибками номера – в результате в форме списка будут показаны только лицевые счета с неправильно сгенерированным номером.

§   Обновить номера лицевых счетов - в ходе обновления будут обновлены номера лицевых счетов, согласно установленной маске. Также будут обновлены признаки доверительного управления и вида деятельности из настройки портфеля.

В карточке лицевого счета имеются поля:

§   Код – уникальный код, идентификатор объекта в Системе. При автоматическом создании код записывается следующим образом:

Разряды 1-5: Балансовый счет единого плана счетов 2-го порядка.

Разряды 6-8: Валюта: указывается цифровой код валюты счета в соответствии с ОКВ, при этом валюта «Рубли» обозначается кодом «810».

Разряд 9: Признак доверительного управления: указывается 2 (клиентские операции).

Разряд 10: Дополнительный цифровой код: указывается 0.

Разряды 11-20: Прочие разряды: заполняются в зависимости от учета (содержат порядковый номер счета при его создании в соответствии с установленными настройками).

Представление кода по умолчанию формируется как строка вида: «*****.***.*.*.**********», то есть разряды кода разделены точками (например, код лицевого счета выглядит так: «10603.810.1.1.8140400001»). Если пользователь хочет изменить представление, то в справочнике «Маски лицевого счета» в реквизите «Маска представления счета» следует указать соответствующее представление (допустим, надо убрать все точки: «********************»), и дальше надо указать эту маску для нужного счета в регистре сведений «Настройка аналитики лицевых счетов ЕПС». После этого представление лицевых счетов соответствующего счета обновится в списке автоматически (в результате код будет отображаться как «10603810118140400001»). 

§   Счет ЕПС – выбор счета из единого плана счетов.

§   Портфель - выбирается портфель, для которого будет использоваться этот лицевой счет.

§   Валюта – выбор валюты счета.

 

Дальше для непарных лицевых счетов в разделе «Аналитика» отображается аналитика, установленная в регистре «Настройка аналитики лицевых счетов НФО».

В аналитическом учете на счетах второго порядка, определенных «Списком парных счетов, по которым может изменяться сальдо на противоположное», приведенном в Приложении 4 Положения 486-П, открываются парные лицевые счета. Парные лицевые счета в Системе хранятся в регистре сведений «Парные лицевые счета».

Допускается наличие остатка только на одном лицевом счете из открытой пары: активном или пассивном. В начале отчетного периода операции начинаются по лицевому счету, имеющему сальдо (остаток), а при отсутствии остатка - со счета, соответствующего характеру операции. Если в конце отчетного периода на лицевом счете образуется сальдо (остаток), противоположное признаку счета, то есть на пассивном счете - дебетовое или на активном - кредитовое, то оно должно быть автоматически перенесено бухгалтерской записью на соответствующий парный лицевой счет по учету средств. Если по каким-либо причинам образовалось сальдо (остатки) на обоих парных лицевых счетах, необходимо в конце отчетного периода автоматически перечислить бухгалтерской записью меньшее сальдо на счет с большим сальдо, то есть на конец отчетного периода должно быть только одно сальдо: либо дебетовое, либо кредитовое на одном из парных лицевых счетов. Эта операция в Системе проводится документом «Закрытие парных счетов ЕПС» при выполнении одноименной регламентной операции. Периодичность выполнения закрытия парных лицевых счетов настраивается для каждого фонда, по которому ведётся учет по единому плану счетов, в документе «Учетная политика» на вкладке «Периодичность», странице «Закрытие периода».

Счета без признака счета – это те счета, по которым не стоит однозначно, что счет может быть только активным или пассивным. Это, к примеру, счета определения финансового результата. Счета, не имеющие признака счета, введены для контроля над своевременным отражением операций, подлежащих завершению в течение дня. По состоянию на конец дня остатков по счетам без указания признака счета быть не должно.

Из формы элемента «Лицевой счет ЕПС» можно перейти в парный ему активный или пассивный счет. Для этого в верхней части формы предусмотрены гиперссылки «Парные лицевые счета (Счет активный)» и «Парные лицевые счета (Счет пассивный)».

После нажатия кнопки «Записать и закрыть», Система сохраняет введенные данные в справочнике «Лицевые счета ЕПС» и закрывает окно ввода информации.
